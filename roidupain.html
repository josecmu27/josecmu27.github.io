Roi Du Pain

For this multi-phase course project, I helped build Roi du Pain, an online ordering system for a bakery using Rails 7.0.4 and Ruby 3.1.4 with a strong emphasis on Model-View-Controller (MVC) design and Test-Driven development (TDD).


Models, Business Logic, and Authorization

I implemented the domain models based on the provided ERD and added the required business logic and validations. I wrote comprehensive unit tests for every model method and used SimpleCov to ensure 100% test coverage. I also implemented authentication and role-based authorization so customers, bakers, shippers, and managers each had the correct permissions. Furthermore, I implemented two service objects: a baking service (daily baking lists), and a shipping service (shipping cost calculator).

Controllers, Views, and API's

Building on the model layer, I focused on the controllers and views to have a proper front-end. The course supplied a full set of controller specs and view test, thus I coded the controllers to pass all tests at 100% coverage and verified the view's end-to-end behavior with Cucumber scenarios. I also created API endpoints, returning the exact JSON shapes specified. For the front end, I implemented the key customer-facing views (catalog, search, cart, checkout) while following design principles.

Process & Iteration

The project was structured with incremental checkpoints to encourage a professional workflow. Early milestones focused on building working API endpoints, followed by full controller and view implementation with 100% test coverage. Each phase required delivering features on time, integrating them with the existing system, and verifying them against both unit and end-to-end tests. This process reinforced best practices in test-driven development, incremental delivery, and proper version control via git.
