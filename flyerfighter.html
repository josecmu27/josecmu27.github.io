Overview

FlyerFighters was a semester long project for Advanced Game Studio at Carnegie Mellon, built with a cross disciplinary team of five programmers, artists, and a product manager. The game is a firefighter themed VR experience set in a futuristic city, where players fly around and rescue civilians using a traversal mechanic built around fire extinguishers. That mechanic came out of early prototyping, where flying by extinguisher blast turned out to be the most fun of several traversal ideas we tried, and it became the foundation the rest of the game was built around.

My contributions covered programming, UI, and level design. This project gave me experience inside a real cross disciplinary team looks like, coordinating systems work with artists and a product manager rather than building solo.

Systems

Game Manager and Save System

I built the Game Manager that handled core systems across the whole game: game state, player statistics, and save data. This is what let players leave a level mid session and pick back up where they left off as they explored the city and rescued civilians, rather than losing progress between play sessions.

Object Cutting Mechanic

Players can cut through obstacles like wood beams using VR motion controls. Each cuttable object starts as a solid piece with its pre-cut fragments hidden underneath, gravity disabled and physics set to kinematic so they hold in place. A set of cut zone triggers sits along the cut line, and each one the player's weapon passes through gets cleared from a counter on the object.

Once every cut zone has been triggered, the object switches its fragments to a walkable physics layer, enables gravity and physics simulation so they fall naturally, and removes the trigger zones immediately so they cannot be hit again. The fragments clean themselves up a few seconds later so they do not accumulate in the scene.

Implementation Note: This is not real time geometry cutting, it is a reveal built on pre-authored fragments and a trigger counter. That kept it fast enough to run smoothly in VR and simple enough to build in the time we had, at the cost of only supporting cuts along a predefined line rather than an arbitrary one.
